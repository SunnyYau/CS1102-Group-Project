Image Compression Algorithms

Based on the four file formats, we found that those formats are produced using these common image compression algorithms: DCT, LZW, Deflate (Combined by using LZ77 and Huffman coding)

Algorithm Well-Suited for Lossyness Compression Ratio Compression Speed
DCT Continuous-tone photographic images with smooth color transitions and complex spatial information Lossy compression, Removes high-frequency information High compression ratios achievable Slow when compressing larger size files
LZW Image files containing a lot of repetitive data, which is common with monochrome images Lossless compression, Preserves all original image information Moderate compression ratios Slow when file data is large and not repetitive
DEFLATE General-purpose compression, as it strikes a good balance between compression ratio, encode speed and decode speed on most data types Lossless compression, Preserves all original image information Moderate compression ratios Generally fast and efficient

Considerations for using which Image Compression Algorithm

Image Quality Requirements: If preserving image quality without introducing compression artifacts is essential, lossless compression algorithms like LZW or Deflate (combined LZ77 and Huffman coding) are more suitable. This is important in applications such as medical imaging (e.g., X-ray), where accurate representation of visual data is critical.

Compression Efficiency: If achieving high compression ratios while accepting some loss of image quality is acceptable, a lossy compression algorithm like DCT (used in JPEG) is more suitable. This is common when there are limited storage space or bandwidth constraints, where reducing file size is crucial for storage or transmission.

Compatibility: Consider the compatibility of the compression algorithm with existing systems, software, and standards. For example, if we need to integrate with platforms that support specific image formats (e.g., JPEG, PNG, GIF), we should use compression algorithms commonly associated with those formats.

Resource Constraints: If we are working with devices or environments with limited storage space or bandwidth, we should optimize for efficient compression algorithms that strike a balance between compression ratio and computational resources.
